# prompt_contribute_button.md — Auto-Contribute Feature

## Overview
Add a **"Contribute"** button to each suggestion card. When tapped, it automatically makes a small
meaningless file change in that suggestion's repo via the **GitHub Contents API** — no git, no
terminal, no extra setup. Just one tap → green square added to the graph. 🟩

---

## How It Works (Flow)

```
User taps "Contribute" on a suggestion card
        │
        ▼
Check if `.greener/log.md` exists in that repo
        │
        ├── EXISTS → fetch its current content + SHA
        │
        └── NOT EXISTS → prepare to create it fresh
        │
        ▼
Append a new timestamped line to the file content
        │
        ▼
PUT to GitHub Contents API (create or update)
        │
        ▼
✅ Success → show green checkmark + "Contributed!" on card
❌ Fail    → show error snackbar with reason
```

---

## The File Being Changed

**File path in repo:** `.greener/log.md`

**File content format:**
```markdown
# Greener Log
<!-- Auto-generated by GitHub Greener app -->

- 2026-03-27 14:32 IST — keep the streak alive 🟩
- 2026-03-26 09:15 IST — keep the streak alive 🟩
- 2026-03-25 21:44 IST — keep the streak alive 🟩
```

Each tap **appends one new line** with the current timestamp.
This creates a real file change → real commit → green square on graph. ✅

## Service to Build

Create `lib/services/contribute_service.dart`:

```dart
class ContributeService {

  /// Main method — call this on button tap
  Future<ContributeResult> contribute({
    required String owner,       // GitHub username
    required String repoName,    // repo to commit to
    required String pat,         // GitHub PAT from secure storage
  });
}

class ContributeResult {
  final bool success;
  final String? commitUrl;  // link to the commit on GitHub
  final String? error;
}
```

## GitHub Contents API — Step by Step

### Base URL
```
https://api.github.com/repos/{owner}/{repo}/contents/{path}
```

### Headers (always include)
```
Authorization: bearer <PAT>
Content-Type: application/json
User-Agent: GitHubGreener/1.0
Accept: application/vnd.github+json
```

---

### Step 1 — Check if file exists (GET)

```
GET https://api.github.com/repos/{owner}/{repo}/contents/.greener/log.md
```

**Response 200** → file exists, extract:
- `content` field (base64 encoded current content) → decode it
- `sha` field → needed for the update call

**Response 404** → file doesn't exist yet → will create fresh

---

### Step 2 — Build new file content

```dart
String buildNewContent(String? existingContent, DateTime now) {
  final timestamp = DateFormat('yyyy-MM-dd HH:mm').format(now);
  final newLine = '- $timestamp IST — keep the streak alive 🟩';

  if (existingContent == null || existingContent.isEmpty) {
    return '# Greener Log\n<!-- Auto-generated by GitHub Greener app -->\n\n$newLine\n';
  }

  return '$existingContent\n$newLine\n';
}
```

Then **base64 encode** the new content:
```dart
final encoded = base64Encode(utf8.encode(newContent));
```

---

### Step 3 — Create or Update file (PUT)

```
PUT https://api.github.com/repos/{owner}/{repo}/contents/.greener/log.md
```

**Request body (JSON):**
```json
{
  "message": "chore: update greener log [skip ci]",
  "content": "<base64 encoded file content>",
  "sha": "<sha from GET — omit this field entirely if creating new file>"
}
```

> ⚠️ If creating a new file (404 on GET), do NOT include the `sha` field at all.
> If updating existing file, `sha` is REQUIRED — without it the API returns 409 Conflict.

**Response 200/201** → success, extract `commit.html_url` for the commit link

## ContributeService Full Implementation Guide

```dart
Future<ContributeResult> contribute({
  required String owner,
  required String repoName,
  required String pat,
}) async {
  final filePath = '.greener/log.md';
  final url = 'https://api.github.com/repos/$owner/$repoName/contents/$filePath';
  final headers = {
    'Authorization': 'bearer $pat',
    'Content-Type': 'application/json',
    'User-Agent': 'GitHubGreener/1.0',
    'Accept': 'application/vnd.github+json',
  };

  try {
    // Step 1: GET existing file
    String? existingSha;
    String? existingContent;

    final getResponse = await http.get(Uri.parse(url), headers: headers);

    if (getResponse.statusCode == 200) {
      final data = jsonDecode(getResponse.body);
      existingSha = data['sha'];
      // decode base64 content (GitHub adds newlines, remove them)
      existingContent = utf8.decode(
        base64Decode((data['content'] as String).replaceAll('\n', ''))
      );
    } else if (getResponse.statusCode != 404) {
      return ContributeResult(success: false, error: 'GET failed: ${getResponse.statusCode}');
    }

    // Step 2: Build new content
    final newContent = buildNewContent(existingContent, DateTime.now());
    final encoded = base64Encode(utf8.encode(newContent));

    // Step 3: PUT updated file
    final body = <String, dynamic>{
      'message': 'chore: update greener log [skip ci]',
      'content': encoded,
    };
    if (existingSha != null) body['sha'] = existingSha;

    final putResponse = await http.put(
      Uri.parse(url),
      headers: headers,
      body: jsonEncode(body),
    );

    if (putResponse.statusCode == 200 || putResponse.statusCode == 201) {
      final data = jsonDecode(putResponse.body);
      final commitUrl = data['commit']['html_url'] as String?;
      return ContributeResult(success: true, commitUrl: commitUrl);
    } else {
      return ContributeResult(success: false, error: '${putResponse.statusCode}');
    }

  } catch (e) {
    return ContributeResult(success: false, error: e.toString());
  }
}
```

## Important Notes

- The commit message includes `[skip ci]` to avoid triggering CI/CD pipelines unnecessarily
- The `.greener/` folder prefix keeps the log file visually separate from real project files
- PAT needs `repo` scope (read + write) for this to work on private repos
- For public repos, `public_repo` scope is enough
- Remind user in Settings screen: *"Your PAT needs 'repo' scope for the Contribute feature to work"*
- After a successful contribute, call `contributionsProvider.refresh()` to re-fetch the graph and show the new green square
